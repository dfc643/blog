Deploying Tengine+Redmine+Gitolite on Ubuntu

INTRODUCE

In a project, we need collaboration writing code. Git can help us to manage code, and Gitolite can help us¬†assign permissions for Git, and Redmine can help us to control Gitolite and Git more easier. But, how to deployment a fully project manage system? Please go follow.

PREPARING

Following this post, you will get install something on Ubuntu:

  • The necessary libraries
  • MySQL Server
  • Ruby and Rails
  • Tengine with Passenger
  • Git and Gitolite
  • Redmine

INSTALLATION STEPS

1. Ubuntu Linux server version

In the first, you need install a Ubuntu server version on you server or desktop computer. If your server performance is low, you can do installation on other computer. But remember, two Operating System version and¬†architecture must be same!!!

2. Install and startup MySQL server

apt-get install mysql-server
service mysql start

About more how to install MySQL server on Ubuntu, please read here.

3. Install Ruby

DONOT install ruby and rails from Ubuntu system software source, It will cause failed in later steps!!!¬†I¬†choose to install Ruby and Rails from Launchpad.

# apt-add-repository ppa:brightbox/ruby-ng  // Add Repositiory
# apt-get update                            // 
# apt-get install ruby2.2                   // Install Ruby (version:2.2)

And then, do a test to know Ruby is install correct.

# ruby -v                                   // Show Ruby version
# gem -v                                    // Show Gem version
# readlink /usr/bin/ruby                    // Check Symbolic link
# readlink /usr/bin/gem                     // Check Symbolic link

Then, Install dependent libraries.

# apt-get install ruby2.2-dev
# apt-get install zlib1g-dev
# apt-get install sqlite3 libsqlite3-dev
# apt-get install g++

Now, Ruby installation has been finished.

6. Install Git

Install Git from Ubuntu system software source directly.

apt-get update
apt-get install git¬†git-core

Check Git installation and version. Redmine required git version > 1.7

git version

In the end of Git installation, Add a user for Git to run.

adduser git
passwd -d git
mkdir /home/git
chown -R git:git /home/git

7. Install¬†Redmine

First, create redmine user and group on Ubuntu system.

adduser redmine
passwd -d redmine
mkdir /home/redmine
chown -R redmine:redmine /home/redmine

DONWLOAD: Then, switch to redmine user and download redmine

root# su redmine
redmine$ wget http://www.redmine.org/releases/redmine-3.2.2.tar.gz
redmine$ tar -zxf redmine-3.2.2.tar.gz
redmine$ ln redmine-3.2.2 redmine
redmine$ cd redmine

If your server in China, Modify Gemfile first line to:

source 'https://ruby.taobao.org'

SETUP DATABASE: Then, setup database for redmine. You can setup database by phpMyAdmin, MySQL Command-Line, etc. Remember replace my_password to your redmine database password you want.

CREATE DATABASE redmine CHARACTER SET utf8;
CREATE USER 'redmine'@'localhost' IDENTIFIED BY 'my_password';
GRANT ALL PRIVILEGES ON redmine.* TO 'redmine'@'localhost';

CONFIGURE DATABASE SETTING: Copy config/database.yml.example to config/database.yml:

redmine$ cp config/database.yml.example  config/database.yml

Modify production section, like this:

production:
  adapter: mysql2
  database: redmine
  host: localhost
  username: redmine
  password: "my_password"
  encoding: utf8

8. Download and configure Gitolite plugins for Redmine

1)¬†Install dependencies

root# apt-get install build-essential libssh2-1 libssh2-1-dev cmake libgpg-error-dev

2) Clone the plugin

# Switch user
root# su - redmine

# First git clone Bootstrap Kit
redmine$ cd redmine/plugins
redmine$ git clone https://github.com/jbox-web/redmine_bootstrap_kit.git
redmine$ cd redmine_bootstrap_kit/
redmine$ git checkout 0.2.4

# Then Redmine Git Hosting plugin
redmine$ cd redmine/plugins
redmine$ git clone https://github.com/jbox-web/redmine_git_hosting.git
redmine$ cd redmine_git_hosting/
redmine$ git checkout 1.2.0

3) Create SSH Keys for user running Redmine

redmine$ cd ~
redmine$ mkdir ssh_keys
redmine$ ssh-keygen -N '' -f ssh_keys/redmine_gitolite_admin_id_rsa
redmine$ cp ssh_keys/redmine_gitolite_admin_id_rsa.pub /tmp/

4) Install Gitolite

root# su git
git$ rm -rf ~/.ssh/
git$ mkdir $HOME/bin
git$ source $HOME/.profile
git$ git clone git://github.com/sitaramc/gitolite
git$ gitolite/install -to $HOME/bin
git$ cp /tmp/redmine_gitolite_admin_id_rsa.pub ~/
git$ bin/gitolite setup -pk redmine_gitolite_admin_id_rsa.pub

5) Configure .bashrc for git

git$ vi ~/.bashrc

And then copy following code into it and save it.

# set PATH so it includes user private bin if it exists
if [ -d "$HOME/bin" ] ; then
  PATH="$PATH:$HOME/bin"
fi

6) Set Gitolite accept hook keys

git$ vi (or nano) .gitolite.rc

## Look for GIT_CONFIG_KEYS and make it look like :
GIT_CONFIG_KEYS  =>  '.*',

## Enable local code directory
LOCAL_CODE       =>  "$ENV{HOME}/local"

## then save and exit

7) Configure sudo

root# visudo

Add two line, save it:

redmine        ALL=(git)      NOPASSWD:ALL
git            ALL=(redmine)  NOPASSWD:ALL

8) Add Gitolite server in known_hosts list

Make sure that Redmine user has Gitolite server in his known_hosts list. This is also a good check to see if Gitolite works.

It should not ask you for a password.

Also the SSH config file for Redmine user (<redmine user home>/.ssh/config) is not needed anymore! Remove it if exists!

root$ su - redmine
redmine$ ssh -i ssh_keys/redmine_gitolite_admin_id_rsa git@localhost info
# accept key

You should get something like that :

hello redmine_gitolite_admin_id_rsa, this is gitolite v2.3.1-0-g912a8bd-dt running on git 1.7.2.5
the gitolite config gives you the following access:
    R   W  gitolite-admin
    @R_ @W_ testing

Or

hello redmine_gitolite_admin_id_rsa, this is git@dev running gitolite3 v3.3-11-ga1aba93 on git 1.7.2.5
    R W  gitolite-admin
    R W  testing

If you get one of these messages, you‚Äôre on the right way üòâ

9. Initial Redmine and Plugins

1) Initial database and plugins

root# cd /home/redmine/redmine
root# bundle install --without development test
root# rake generate_secret_token
root# RAILS_ENV=production rake db:migrate
root# RAILS_ENV=production rake redmine:load_default_data
root# bundle exec rake redmine:plugins:migrate RAILS_ENV=production NAME=redmine_git_hosting

2) Create upload directory and give write permissions

root# cd /home/redmine/redmine
root# mkdir -p tmp tmp/pdf public/plugin_assets
root# chown -R redmine:redmine files log tmp public/plugin_assets
root# chmod -R 755 files log tmp public/plugin_assets

10. Configure Tengine site

Creating a new site configuration file in Tengine with following code:

passenger_root /var/lib/gems/2.2.0/gems/passenger-5.0.28;
passenger_ruby /usr/bin/ruby2.2;

server {
        listen 80;
        server_name redmine.youdomain.com;

        root /home/redmine/redmine/public;

        access_log    /var/log/nginx/redmine.log;
        error_log     /var/log/nginx/redmine.log;

        client_max_body_size 100M;
        passenger_enabled on;
}

And restart your tengine server access http://redmine.yourdomain.com do final configuration.

11. Final Configuration

  1. Enable Xitolite repositories in Administration -> Settings -> Repositories
  2. Configure plugin settings in Administration -> Redmine Git Hosting, specially :
    • SSH Keys path
    • Temp dir path
    • Access urls
    • Hooks url (This url will be called by our Gitolite hook to trigger repository view refresh so be sure it‚Äôscallable)
    • Hooks install (This may overwrite your existing Gitolite hooks)
  3. Check your installation in Administration -> Redmine Git Hosting Config Checks tab.
  4. Set some permissions on Administration -> Roles page, particularly if you want users to be able to create SSH keys (see below).

Before configuring the plugin you should take a look at this : Repositories Storage Configuration Strategy.
