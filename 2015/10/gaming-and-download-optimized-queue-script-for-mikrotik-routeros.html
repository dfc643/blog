Optimized queue script for MikroTik RouterOS

Network Norckon 2015-10-18 4750℃ 0评论

INTRODUCE

Have you complained about a large gaming latency when somebody downloading file or watching video. Some stronger downloader will takes all the network into busy. But any solution to solve? Yes, this article will told you how to use queue control or manage your network traffic.

ENVIRONMENT

  • Network: China Telecom ADSL (up:760kbps down:10mbps)
  • Router: MikroTik RouterOS 3.30
  • Scale: About 5~10 devices
  • Description: Gaming takes a large lag when someone takes downloading
  • Local Network: 192.168.88.0/24 (LAN) 10.0.0.0/24 (PPPoE Server)

OPERATING STEPS

Step 1: Create a basic address list, used for describe your network.

/ip firewall address-list
add list=Local-Network address=192.168.88.0/24
add list=Local-Network address=10.0.0.0/24
add list=HOST-NET address=192.168.100.0/24
  • List Local-Network: your router controlled network
  • List HOST-NET: server or other device in your network controlled network

NOTICE: If your network doesn’t have any server or device, replace with your router network.

Step 2: Mangle your network traffic, used for queue your network traffic.

/ip firewall mangle
add action=mark-connection chain=prerouting comment=\
    "Mangled for queue tree by port and size (Upload)" disabled=no \
    new-connection-mark=game_up_conn packet-size=0-512 passthrough=yes \
    protocol=udp src-address-list=Local-Network
add action=mark-packet chain=prerouting comment="" connection-mark=\
    game_up_conn disabled=no new-packet-mark=game_up packet-size=0-512 \
    passthrough=no src-address-list=Local-Network
add action=mark-connection chain=prerouting comment="" disabled=no \
    new-connection-mark=icmp_up_conn passthrough=yes protocol=icmp \
    src-address-list=Local-Network
add action=mark-packet chain=prerouting comment="" connection-mark=\
    icmp_up_conn disabled=no new-packet-mark=icmp_up passthrough=no protocol=\
    icmp src-address-list=Local-Network
add action=mark-connection chain=prerouting comment="" disabled=no dst-port=\
    80,8080,443,53 new-connection-mark=web_up_conn packet-size=0-1024 \
    passthrough=yes protocol=tcp src-address-list=Local-Network
add action=mark-packet chain=prerouting comment="" connection-mark=\
    web_up_conn disabled=no new-packet-mark=web_up packet-size=0-1024 \
    passthrough=no src-address-list=Local-Network
add action=mark-connection chain=prerouting comment=\
    "Mangled for queue tree by packet size (Upload)" disabled=no \
    new-connection-mark=small_up_con packet-size=0-512 passthrough=yes \
    src-address-list=Local-Network
add action=mark-packet chain=prerouting comment="" connection-mark=\
    small_up_con disabled=no new-packet-mark=small_up packet-size=0-512 \
    passthrough=no src-address-list=Local-Network
add action=mark-connection chain=prerouting comment="" disabled=no \
    new-connection-mark=big_up_con packet-size=513-1500 passthrough=yes \
    src-address-list=Local-Network
add action=mark-packet chain=prerouting comment="" connection-mark=big_up_con \
    disabled=no new-packet-mark=big_up packet-size=513-1500 passthrough=no \
    src-address-list=Local-Network
add action=mark-connection chain=prerouting comment=\
    "Mangled for queue tree by port and size (Download)" disabled=no \
    new-connection-mark=game_down_conn packet-size=0-512 passthrough=yes \
    protocol=udp
add action=mark-packet chain=prerouting comment="" connection-mark=\
    game_down_conn disabled=no new-packet-mark=game_down packet-size=0-512 \
    passthrough=no
add action=mark-connection chain=prerouting comment="" disabled=no \
    new-connection-mark=icmp_down_conn passthrough=yes protocol=icmp
add action=mark-packet chain=prerouting comment="" connection-mark=\
    icmp_down_conn disabled=no new-packet-mark=icmp_down passthrough=no \
    protocol=icmp
add action=mark-connection chain=prerouting comment="" disabled=no \
    new-connection-mark=web_down_conn packet-size=0-1024 passthrough=yes \
    protocol=tcp src-port=80,8080,443,53
add action=mark-packet chain=prerouting comment="" connection-mark=\
    web_down_conn disabled=no new-packet-mark=web_down packet-size=0-1024 \
    passthrough=no
add action=mark-connection chain=prerouting comment=\
    "Mangled for queue tree by packet size (Downlaod)" disabled=no \
    new-connection-mark=small_down_con packet-size=0-512 passthrough=yes
add action=mark-packet chain=prerouting comment="" connection-mark=\
    small_down_con disabled=no new-packet-mark=small_down packet-size=0-512 \
    passthrough=no
add action=mark-connection chain=prerouting comment="" disabled=no \
    new-connection-mark=big_down_con packet-size=513-1500 passthrough=yes
add action=mark-packet chain=prerouting comment="" connection-mark=\
    big_down_con disabled=no new-packet-mark=big_down packet-size=513-1500 \
    passthrough=no

The above command will divide network traffic into 2 main regions and 5 sub-regions. And each region divide into upload and download traffic.

Main Regions Sub Regions
Application Layer Gaming
ICMP
Web Browsing
​Network Layer Big Packet
Small Packet

Step 3: Create  three queue types, used control network traffic.

/queue type
add kind=pcq name=pcq-down pcq-classifier=dst-address pcq-limit=50 pcq-rate=\
    10000000 pcq-total-limit=2000
add kind=pcq name=pcq-up pcq-classifier=src-address pcq-limit=50 pcq-rate=\
    760000 pcq-total-limit=2000
add kind=red name=red-force red-avg-packet=1000 red-burst=20 red-limit=60 \
    red-max-threshold=50 red-min-threshold=10
  • Type pcq-down: Control downlink traffic
  • Type pcq-up: Control uplink traffic
  • Type red: Control network traffic by drop packets

NOTICE: Replace pcq-rate value in pcq-down to your downlink speed in bit, and replace pcq-rate value in pcq-up to your uplink speed in bit.

Step 4: Create queue tree, used for control and manage network traffic.

/queue tree
add burst-limit=0 burst-threshold=0 burst-time=0s disabled=no limit-at=0 \
    max-limit=0 name=Download parent=global-in priority=8
add burst-limit=0 burst-threshold=0 burst-time=0s disabled=no limit-at=0 \
    max-limit=0 name=small packet-mark=small_down parent=Download priority=2 \
    queue=pcq-down
add burst-limit=0 burst-threshold=0 burst-time=0s disabled=no limit-at=0 \
    max-limit=<90%_DOWNLINK> name=big packet-mark=big_down parent=Download priority=8 \
    queue=red-force
add burst-limit=0 burst-threshold=0 burst-time=0s disabled=no limit-at=0 \
    max-limit=0 name=web packet-mark=web_down parent=Download priority=7 \
    queue=pcq-down
add burst-limit=0 burst-threshold=0 burst-time=0s disabled=no limit-at=0 \
    max-limit=0 name=game packet-mark=game_down,icmp_down parent=Download \
    priority=1 queue=pcq-down
add burst-limit=0 burst-threshold=0 burst-time=0s disabled=no limit-at=0 \
    max-limit=0 name=Upload parent=global-out priority=8
add burst-limit=0 burst-threshold=0 burst-time=0s disabled=no limit-at=0 \
    max-limit=<20%_UPLINK> name=big- packet-mark=big_up parent=Upload priority=8 \
    queue=pcq-up
add burst-limit=0 burst-threshold=0 burst-time=0s disabled=no limit-at=0 \
    max-limit=0 name=web- packet-mark=web_up parent=Upload priority=7 \
    queue=pcq-up
add burst-limit=0 burst-threshold=0 burst-time=0s disabled=no limit-at=0 \
    max-limit=0 name=small- packet-mark=small_up parent=Upload priority=2 \
    queue=pcq-up
add burst-limit=0 burst-threshold=0 burst-time=0s disabled=no limit-at=0 \
    max-limit=0 name=game- packet-mark=game_up,icmp_up parent=Upload \
    priority=1 queue=pcq-up

Now, your queue tree will look like this.

NOTICE: Remember replace <90%_DOWNLINK> and <20%_UPLINK> to your values !

After all the steps, your network get stable now. You can get best gaming experience when someone is downloading. But if someone using a Thunder downloader, you need block this downloader server IP addresses.

ADDITIONAL STEPS

Using UDP white-list to block stronger downloader

1. Create a address list of IDC hosts which you won’t block it.

/ip firewall address-list
add address=14.17.0.0/16 list=IDC-NODE
add address=14.18.0.0/16 list=IDC-NODE
add address=14.19.0.0/16 list=IDC-NODE

2. Create 2 rules in filter, only allow white-list UDP connection pass.

/ip firewall filter
add action=accept chain=forward comment=\
    "Allow Udp ports in white-list" disabled=no dst-address-list=IDC-NODE
add action=drop chain=forward comment=\
    "Drop Udp ports not in white-list or big packets" disabled=no \
     protocol=udp dst-address-list=Local-Network
