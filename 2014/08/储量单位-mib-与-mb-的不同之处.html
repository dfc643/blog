储量单位 MiB 与 MB 的不同之处

Operating System Norckon 2014-08-17 1370℃ 0评论

MB 是我们常见的一种计算机储量单位，经常出现于 Windows 操作系统当中，但是另外一种类似的储量单位却不是那么熟悉。在储量单位 Megabytes (MB) 与 Mebibytes (MiB) 之间却存在一些细微的差别，这种差别在 Windows 中几乎没有表现出来，因为 Windows 将他们都视为 MiB 单位使用（混用）；而这种差别在 Linux 中将会有明显的体现。除了 MB 与 MiB 之外，还存在着 KB KiB GB GiB TB TiB 等等。

这种单位的换算关系大概可以体现为下面形式：

  • MiB = Mebibyte = 1024 KiB
  • KiB = Kibibyte = 1024 Bytes
  • MB = Megabyte = 1,000 KB
  • KB = Kilobyte = 1,000 Bytes

对于这两种单位而言，MiB 采用 2 为底数的乘积运算，而 MB 则常用了 10 为底数的乘积运算。至于详细的说明，请参见 NIST 的数据图表。

虽说两者之间存在一些细微区别，而实质上却差异不是很大，在 Windows 中才会出现混用的情况。因此经常会出现一些有趣的存储容量或者文件大小的对比。参考维基百科。

比如说，在 Windows XP 的文件属性对话框中，会将一个 220 字节的文件显示成 1.00MB ，而会把 106 (1,000,000) 直接的文件显示成 976KB 。

在苹果的 Mac OS X 10.6 中，换种说话，会将一个 106 字节的文件正确地报告为 1.00MB 大小。

在 Linux 中

当你将一个 4GB 的 U 盘接入 Linux 的电脑中时，打开磁盘管理后会报告该磁盘的容量。仔细查看你会发现其报告两种储量单位，一种为 GB 另一种则为 GiB，Linux 将 GiB 储量单位作为主要的计量单位。

其中 GB 的单位将会报告该 U 盘的实际容量为 4.0 GB，而 GiB 的单位将会报告该 U 盘的实际容量为 3.8 GiB。后者单位报告的容量与 Windows 所报告容量一致。

在 Windows 中

同样的，我们将一个 4GB 的 U 盘接入 Windows 电脑中时，打开磁盘管理后仅报告一种储量单位，那就是 GB。但是这与 Linux 中不同，它不会报告该 U 盘有 4GB 容量，而是报告的 3.8GB 容量。这是什么原因？

这是由于 Windows 的储量单位混用引起的。在 Windows 操作系统中，不管是 GB 还是 GiB 统统采用 2 为底数的乘积计算方式计算，也就是 Linux 中 GiB 的计算方式。

简单的看待，在 Windows 操作系统中，我们可以将所有的 KB MB GB TB 视为 KiB MiB GiB TiB 储量单位。

总结

虽然这两种单位存在区别不是很大，尽管最终计算出来容量误差也不大，甚至 Windows 都出现单位混用情况，但是不推荐将两者混用，两者之间存在实质性区别。

若觉得底数乘积较难理解，那么可以简单的理解成，MB 单位的进率为 1000，而 MiB 的进率为 1024。
