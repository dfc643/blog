Windows网络终极修复办法

Network Norckon 2014-04-01 2410℃ 0评论

在正常使用Windows操作系统的时候，我们可能会由于一些意外的情况而遇到网络故障，这种故障可大可小。稍微小的一点的则是不妨碍正常使用而只是一些显示错误，而稍微大一点的错误则会引起广域网甚至局域网的连通出错。

北极光今天就遇到了一个不算大的Windows网络错误，不会干涉到平时的正常上网行为，但是你修改过网卡的IP地址之后可能会出现一些问题。比如说，重启后同时出现许多个IP地址，又比如从静态IP修改为动态IP时必须禁用再启用网卡才能获取到IP地址。

这给经常要进行网络作业的北极光来说，带来的不仅仅是视觉上的麻烦，而更多的是使用的不便利。为了解决这个问题，北极光从简单到复杂进行了多项测试，最终经过2小时的倒腾终于解决了这个看似不大的网络问题。

北极光的解决思路是这样的，对于一个问题，首先我们需要对这个问题行进一定的分析（由于北极光使用的系统是英语作业系统，所以文中提及的选项均以英文为准）：

这个错误有些复杂，北极光的电脑在有网络错误的时候，是可以正常访问互联网与局域网的。只是无法为网络适配器配置一个IP地址。在打开TCP/IP配置界面的时候会弹出这样的错误提示「in order to configure tcp/ip you must install and enable a network adaptor card」。

北极光的电脑明明安装了并且启用了网络适配卡，为什么还是会提示这样的错误信息呢？根据北极光的平时的经验，一般重新安装网卡驱动即可解决问题。

而事实上并没有这么简单，在北极光重装完有线网卡的驱动之后，发现有线网卡的TCP/IP可以打开了，而其他网卡的则依然不能打开。于是北极光就卸载了所有的网络卡驱动程序，然后一一重新安装驱动。这样所有网卡的TCP/IP配置界面均可打开，但是问题并没有这么简单。

虽说现在可以打开网卡的TCP/IP配置界面，但是由于一些错误的存在，导致无法正确的配置静态IP地址或者动态IP地址。当北极光给有线网卡配置上IP地址之后，确实可以使用。但是当你再次打开有线网卡的TCP/IP配置界面的时候，却惊奇的发现默认选中的是自动获取IP地址！而且点击OK保存后依然是之前设置的静态IP地址，根本没法自动获取IP地址！

于是北极光就尝试着改了多个静态IP地址，然后重启计算机后再打开有线网卡的TCP/IP配置界面，这个时候还是和先前一模一样的，选项默认选中的是自动获取，但是实际上的IP地址并不是自动获取的。依然还是手动设置的IP地址。并且！并且之前设置的所有IP地址全部并且显示在这张有线网卡上。也就是说这张有线网卡拥有了多个静态IP地址！并不符合常理。

既然是这样的话，北极光就开始围绕TCP/IP展开调查。起初北极光想到了卸载TCP/IP并且重新安装，但是发现默认不会有卸载按钮。于是北极光就放弃了这个想法，转而使用号称无敌的网络修复工具「Complete Internet Repair」，事实上北极光使用了多次这个工具依然无法解决目前的问题！

于是北极光并没有就此放弃修复问题。北极光经过几十分钟的资料寻找，找到了一个微软官方的TCP/IP修复工具，但是并没有这么简单。当北极光下载好这款FixIt工具之后，打开并修复完成重启完计算机之后，问题依然存在。不过北极光还是认为问题出在TCP/IP协议上。

又经过一段时间的搜索，北极光在谷歌上发现一篇很牛的文章，这篇文章里面介绍了如何重新安装TCP/IP协议。北极光决定好了，如果这次还修复不好就放弃修复了，但是按照文章的内容操作之后发现，可以正常的在静态IP和自动IP之间切换了，而且也不会出现其他的问题了。没错简单地说就是恢复了正常的样子。

那么方法是就是这样的：

1、首先打开命令提示符，传说中的黑窗口。
2、然后输入netsh int ip reset 并回车。
3、然后右击某网卡打开网络连接属性，选中TCP/IP v4（卸载不可用）。
4、点击「安装」按钮，选择协议，再点击从磁盘安装。
5、在c:\Windows\inf目录里面随便选择一个inf文件点击确认。
6、然后弹出窗口选择 TCP/IP v4，再确认。
7、最后回到命令提示符，输入 netsh winsock reset 回车并重启。

重启后所有的问题便解决了，上面的方法主要是先清空 IP 配置，然后再重新安装 TCP/IP 协议，再清空 winsock 配置。可以说得上是给网络来了个大初始化，如果这样依然不能正常使用的话，那么北极光也没有什么好的办法了，请向最好的老师谷歌和百度请教吧。

To repair windows 7 network’s final way:

1. Open command-line window.
2. Run “netsh int ip reset” command.
3. Right click on your network and select TCP/IP v4 (Uninstall unavailable)
4. Click “Install” button and select “Protocol” and click “Hard Disk…”
5. Select a inf file from “C:\Windows\inf” and click OK
6. Double click on TCP/IP v4.
7. Turn back to the command-line, run “netsh winsock reset” and reboot.

This can repair your TCP/IP! have fun!
