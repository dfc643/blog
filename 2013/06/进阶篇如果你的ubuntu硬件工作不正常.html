[进阶篇]如果你的Ubuntu硬件工作不正常

Operating System Norckon 2013-06-15 1976℃ 0评论

警告：本文操作属于进阶篇，所以难度可能具有一点，那么如果不熟悉Linux基本操作的用户，建议不要轻易尝试，以免损坏您的计算机！

在安装完Ubuntu后，不是所有的电脑都能正常工作，况且Ubuntu对笔记本的支持就不好，所以我们需要进行进一步的优化来让其正常工作，十分简单，只需要自己制作一个脚本就能完成了。

首先，你需要安装你的显卡驱动，如果你是AMD加INTEL双显卡的话，那么请参考我之前的那篇文章来完成安装，如果是其他显卡如NVIDIA显卡的话，请自行谷歌一篇比较好的文章来完成显示卡的安装。显示卡的不正确安装可能导致多方面的问题出现。

在你安装完图形卡驱动后，你只需要安装一个powertop软件，这个软件必须在root权限里面运行，所以需要用到sudo。

安装完毕之后，输入sudo powertop -html然后就会生成一个HTML格式的报告文件。双击使用浏览器打开这个文件后，鼠标单击Tuning。这是一个建议栏目，里面包含了优化的脚本，将脚本一条接一条地粘贴到一个记事本文件里面去。注意，每一条要换行一次，其中有些命令可能会提示找不到软件，这个时候你需要去安装相应地软件包。将这个文件保存为一个扩展名为.sh的文件，然后存在自己喜欢的地方。一下是一个范例：

#################################################################
#UBUNTU POWERSAVING SCRIPT for HP G4-1014TX by dfc643
#################################################################

#Turn OFF NMI Watchdog
echo '0' > '/proc/sys/kernel/nmi_watchdog';
#VM writeback timeout
echo '1500' > '/proc/sys/vm/dirty_writeback_centisecs';
#Wireless Power Saving for interface wlan0
iw dev wlan0 set power_save off
#Enable Audio codec power management
echo '1' > '/sys/module/snd_hda_intel/parameters/power_save';
#Enable SATA link power management for /dev/sda
echo 'min_power' > '/sys/class/scsi_host/host0/link_power_management_policy';
#Autosuspend for USB device USB Optical Mouse [1-1.1]
#鼠标可以不开启省电，不然就和无线鼠标一样了，得按一下才能唤醒
#echo 'auto' > '/sys/bus/usb/devices/1-1.1/power/control';
#Autosuspend for USB device USB Multimedia Keyboard [Lite-On Technology Corp.]
echo 'auto' > '/sys/bus/usb/devices/2-1.1/power/control';
#Runtime PM for PCI Device Intel Corporation Core Processor Integrated Graphics Controller
echo 'auto' > '/sys/bus/pci/devices/0000:00:02.0/power/control';
#Runtime PM for PCI Device Intel Corporation Core Processor QPI Physical 0
echo 'auto' > '/sys/bus/pci/devices/0000:7f:02.1/power/control';
#Runtime PM for PCI Device Intel Corporation Core Processor QPI Link 0
echo 'auto' > '/sys/bus/pci/devices/0000:7f:02.0/power/control';
#Runtime PM for PCI Device Intel Corporation Core Processor QuickPath Architecture System Address Decoder
echo 'auto' > '/sys/bus/pci/devices/0000:7f:00.1/power/control';
#Runtime PM for PCI Device Intel Corporation 5 Series/3400 Series Chipset PCI Express Root Port 1
echo 'auto' > '/sys/bus/pci/devices/0000:00:1c.0/power/control';
#Runtime PM for PCI Device Intel Corporation 5 Series/3400 Series Chipset PCI Express Root Port 2
echo 'auto' > '/sys/bus/pci/devices/0000:00:1c.1/power/control';
#Runtime PM for PCI Device Intel Corporation 5 Series/3400 Series Chipset High Definition Audio
echo 'auto' > '/sys/bus/pci/devices/0000:00:1b.0/power/control';
#Runtime PM for PCI Device Intel Corporation 5 Series/3400 Series Chipset USB2 Enhanced Host Controller
echo 'auto' > '/sys/bus/pci/devices/0000:00:1a.0/power/control';
#Runtime PM for PCI Device Intel Corporation 5 Series/3400 Series Chipset HECI Controller
echo 'auto' > '/sys/bus/pci/devices/0000:00:16.0/power/control';
#Runtime PM for PCI Device Intel Corporation Core Processor Reserved
echo 'auto' > '/sys/bus/pci/devices/0000:7f:02.3/power/control';
#Runtime PM for PCI Device Intel Corporation Core Processor PCI Express x16 Root Port
echo 'auto' > '/sys/bus/pci/devices/0000:00:01.0/power/control';
#Runtime PM for PCI Device Intel Corporation Core Processor DRAM Controller
echo 'auto' > '/sys/bus/pci/devices/0000:00:00.0/power/control';
#Runtime PM for PCI Device Intel Corporation 5 Series/3400 Series Chipset PCI Express Root Port 3
echo 'auto' > '/sys/bus/pci/devices/0000:00:1c.2/power/control';
#Runtime PM for PCI Device Intel Corporation Core Processor QuickPath Architecture Generic Non-core Registers
echo 'auto' > '/sys/bus/pci/devices/0000:7f:00.0/power/control';
#Runtime PM for PCI Device Intel Corporation Core Processor Reserved
echo 'auto' > '/sys/bus/pci/devices/0000:7f:02.2/power/control';
#Runtime PM for PCI Device Intel Corporation 82801 Mobile PCI Bridge
echo 'auto' > '/sys/bus/pci/devices/0000:00:1e.0/power/control';
#Runtime PM for PCI Device Intel Corporation Mobile 5 Series Chipset LPC Interface Controller
echo 'auto' > '/sys/bus/pci/devices/0000:00:1f.0/power/control';
#Runtime PM for PCI Device Intel Corporation 5 Series/3400 Series Chipset 4 port SATA AHCI Controller
echo 'auto' > '/sys/bus/pci/devices/0000:00:1f.2/power/control';
#Runtime PM for PCI Device Intel Corporation 5 Series/3400 Series Chipset SMBus Controller
echo 'auto' > '/sys/bus/pci/devices/0000:00:1f.3/power/control';
#Runtime PM for PCI Device Intel Corporation 5 Series/3400 Series Chipset Thermal Subsystem
echo 'auto' > '/sys/bus/pci/devices/0000:00:1f.6/power/control';
#Runtime PM for PCI Device Advanced Micro Devices [AMD] nee ATI Seymour [Radeon HD 6400M/7400M Series]
echo 'auto' > '/sys/bus/pci/devices/0000:01:00.0/power/control';
#Runtime PM for PCI Device Atheros Communications Inc. AR9285 Wireless Network Adapter (PCI-Express)
echo 'auto' > '/sys/bus/pci/devices/0000:02:00.0/power/control';
#Runtime PM for PCI Device Realtek Semiconductor Co., Ltd. RTL8101E/RTL8102E PCI Express Fast Ethernet controller
echo 'auto' > '/sys/bus/pci/devices/0000:03:00.0/power/control';
#Runtime PM for PCI Device Realtek Semiconductor Co., Ltd. RTS5209 PCI Express Card Reader
echo 'auto' > '/sys/bus/pci/devices/0000:04:00.0/power/control';
#Runtime PM for PCI Device Intel Corporation 5 Series/3400 Series Chipset USB2 Enhanced Host Controller
echo 'auto' > '/sys/bus/pci/devices/0000:00:1d.0/power/control';
#Using 'ondemand' cpufreq governor
/sbin/modprobe cpufreq_ondemand > /dev/null 2>&1 ;
echo 'ondemand' > '/sys/devices/system/cpu/cpu0/cpufreq/scaling_governor'; 
echo 'ondemand' > '/sys/devices/system/cpu/cpu1/cpufreq/scaling_governor'; 
echo 'ondemand' > '/sys/devices/system/cpu/cpu2/cpufreq/scaling_governor'; 
echo 'ondemand' > '/sys/devices/system/cpu/cpu3/cpufreq/scaling_governor';
#Wake-on-lan status for device eth0
ethtool -s eth0 wol d;

最后将这个文件制作成为启动项（用户级别），当你登录你的帐号时，就会应用这个脚本。如果作为服务启动，将无法完全应用脚本。方法是在/home/用户名/.config/autostart/目录新建一个.desktop的文本文件，范例如下：

[Desktop Entry]
Type=Application
Encoding=UTF-8
Version=1.0
Name=系统开机配置程序
Comment=开机配置
Exec=/home/dfc643/.config/startup.sh
X-GNOME-Autostart-enabled=true
Name[zh_CN]=startup.desktop

最后配置一下Grub的选项，来适用你的笔记本的物理按键，就是FN功能键，使用gedit打开/etc/default/grub文件，然后修改为下面两句，然后要更新Grub：

GRUB_CMDLINE_LINUX_DEFAULT="quiet splash"
GRUB_CMDLINE_LINUX="acpi_osi=Linux acpi_backlight=vendor"
#如果是AMD/INTEL双显卡用户或者非单intel显卡用户请使用上面的语句
#如果是单纯INTEL用户使用后无效，请将vendor改为intel
#警告，更改可能造成无法启动，一旦无法启动请在Windows安装Ext2fs修改/boot/grub/grub.cfg
